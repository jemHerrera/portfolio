  • Scriddish

    Personal Project

    Scriddish is a single-page meal planning application made using Vue. The purpose of the app is to allow me to select and save my meals for the next few days, and automatically create a grocery list for those meals readily sorted out automatically.

    Scriddish

    HTML / SCSS / Vue3 / Vue Router

    View Project Github Figma

    About Scriddish

    If you much prefer to cook meals at home, then you must understand my struggles. Cooking at home on a consistent basis is a lot harder than it sounds. It requires tremendous amount of dedication, planning and mental effort. This is why most prefer eating out or ordering food online. Here is an example of my usual grocery shopping routine:

    First I have to decide which set of meals to cook for the next few days. This means I have to spend time researching recipes online or remembering recipes that I already know. The next step is to list down ingredients for grocery shopping, but I have to make sure not to add ingredients that already exist in the pantry. I also need to make sure that my grocery list is sorted by meats, produce, condiments, sauces, etc. so I don’t have to waste time going back and forth between different sections and aisles in the grocery store.

    I had to keep on repeating these tasks almost twice a week and I find it quite time consuming. So it made me think, Why not build a tool that can ease up or automate some of these repetitive tasks for the sake of my inner peace? And thus Scriddish was born..

    Scriddish is one of my personal projects, a Single Page Application that I built to help me with my meal planning struggles. I have no idea why I called it Scriddish, but atleast it’s not generic such as “Meal Planner Mock Application”.

    Objectives

    The following are the main functions of the app:

    • Easily choose from a list of recipes
    • Add and remove recipes to a meal plan. A meal plan contains recipes that I want to cook for the next few days
    • Create a sorted list of ingredients for more efficient grocery shopping
    • Search a recipe by name or ingredient
    • View a recipe’s cooking instructions

    Challenges

    Probably the biggest challenge for this project is planning the code architecture. Starting to code from scratch is exciting because there are endless ways to build, but at the same time it can also be quite overwhelming. I had to consider possible features and edge cases on the planning phase to avoid hitting major development walls.

    Aside from building the project itself, my secondary goal is to use this project as an opportunity to practice and learn Vue3’s new Composition API. While learning as I go, I had to experiment on finding better ways to structure my components. It also involves setting up navigation between components and modals and as well as figuring out how to structure recipe data in a way that it’s easy to keep adding more in the future.

    Future Plans

    For the next phase of the project, I am planning to include user features, where accounts can be created. Each individual account can contribute by creating their own recipes, which they can set as private or public, much like github but for recipes. For this I have to learn how to set up a database, as well as learn back-end software development.

    The most essential feature of this website is its ordering system for prescription lenses. Prescription lenses are not structured similarly as the typical e-commerce items such as clothing where each item has only a few varieties of color and size. Prescription lenses are much more complex, where each product is a combination of different kinds of lens type, lens thickness, lens coating, and each has its own range of sphere power and cylinder power. The client wanted to build the prescription selections in a two dimensional grid format, where the Y-axis corresponds to the Sphere power of the lens, while the X-axis corresponds to the Cylinder.

  • Sense of Neumor

    Personal Project

    Sense of Neumor is a simple and quick project I built on the side. The objective is to generate a beautiful 3D shadow, based on the position of a draggable light source. It’s still a work in progress and I’m aiming to add more functionalities as I go.

    Sense of Neumor

    HTML / SCSS / JavaScript

    View Project Github Figma

    About Sense of Neumor

    Sense of Neumor is a simple but fun project. The objective is to feature a new kind of UI style called neumorphism, which is associated with a more realistic shadow effect. Applying neumorphic style to a UI element often involves not just a shadow on one side, but also a highlight on the opposite side. For this to work the background color must be slightly darker than white(#FFF). This creates a realistic 3D effect where a light source is more apparent.

    Dynamic Shadow

    The fun part of this project is to allow user to drag the light source across the screen, which would then change the angle of the shadow on the subject element. The process is about calculating the coordinates of the light source element relative to the subject, computing the angle, then converting the values into css box-shadow syntax.

    Future Plans

    This project is still a work in progress. My plans for the next phase is to allow users to be able to adjust the light intensity, adjust the subject’s size and radius, as well as switch into dark mode. I also would love to include an auto-generated css box where users can copy the css for the given setup.
